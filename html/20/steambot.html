Steam Bot

A steam bot I created for personal use back when I frequently traded. This is a work in progress.

The Problem

It's often times difficult to be available to everyone at all times. This especially goes with strangers online. However, on Steam, I wanted to find a way to be available to people even when I'm not at my computer.

The Solution

By using event listeners, you can set up a bot that is able to respond to certain social actions upon receiving one. This is useful for a few actions in particular:

  • Friend requests: I often receive a lot of these as its the main way users will attempt to interact with me on steam. This could fall under trade requests, random user requests, and phishing attempts.
  • Messages: Upon adding people, I would often receive some sort of message from them. There's no easy way to program a informal conversation with users, so the next best route is to create an interface that users could interact with. This works because I don't have to worry about the bots that added me to phish me, as they won't interact with the interface.

What I did

I created a Steam bot in Javascript.

I looked through Steam's api and learned how I could interact with steam's social features using event listeners.

After logging in, the bot listens for friend requests and certain messages.

  • Upon receiving a friend request, the bot is able to organize the friend into a list and send a message immediately to the user.
  • Upon receiving certain commands from the user, the bot is able to perform certain actions on command.

Additionally, using the interface from my own Steam bot, I attempted to create a bot that would work with Coinbase, allow users to trade Etherum for in-game items, and store user balances. I was creating this for another person but they lost interest in the project before it was completed.

What I want to do next

  • Improve the interface for users.
  • Incorporate a database to help me organize social requests.
